## Final Project Direction
For my final project, I will continue developing my midterm project, **Personal Archive**, into a more complete and interactive portfolio site. The current site introduces my background, inspirations, and selected work through a multi-page visual archive. For the final, I want to expand the site so it feels more dynamic, substantial, and polished, with a stronger emphasis on JavaScript interaction and animation.

## Project Goal
My goal is to turn the midterm into a finished portfolio/archive experience that better represents me and shows stronger front-end development skills. The final version will keep the editorial and archival visual style of the midterm, while adding more content, more interactivity, and more thoughtful transitions between sections.

## Planned Additions and Changes

### 1. Add Interactive Filters
- Add JavaScript-based filters to organize content on the `Work` page.
- Add JavaScript-based filters to organize content on the `Inspirations` page.
- Potential filter groups include category, type, location, or theme.
- Use filtering to make the archive feel more interactive and easier to browse.

### 2. Expand the About Page With Animation
- Add animated transitions between the cities in the timeline.
- Make the movement through Seoul, Xinjiang, Almaty, and Abu Dhabi feel more like a visual story.
- Use scroll-based animation or staged transitions to reveal each city in a more immersive way.

### 3. Add a Contact Page / Contact Section
- Create a complete contact page or section so the site feels finished.
- Include a form where visitors can write a message.
- Connect the form to my email using a simple front-end-friendly solution such as Formspree or another email form service.

### 4. Add More Work and Archive Content
- Expand the `Work` page beyond one project so it feels like a complete portfolio.
- Add more entries to the `Inspirations` archive.
- Continue refining the `About` content to better connect the visual timeline to my personal story.

### 5. Add Lightweight Dynamic Content Management
- Instead of a full backend admin system, I plan to create a lightweight front-end content system.
- Use JavaScript objects, JSON-style data, or local storage to make sections of the site easier to update.
- If possible, create a simple "editor mode" or structured content system so I can add new work or inspiration entries more efficiently without rewriting the page structure each time.
